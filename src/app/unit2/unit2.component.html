Theoretische Grundlagen


Metamodellierung stellt das Fundament für DSMLs dar. Sie gibt die Regeln, Konzepte und Strukturen für diese, auf einer höheren Abstraktionsebene, vor, somit ermöglicht sie es die Elemente dieser Sprache formal, eindeutig und (toolunterstützend) zu spezifizieren. Ohne eine solche Metastruktur wäre eine konsistente und widerspruchsfreie Anwendung eines Modells nicht möglich. Ein präzises Metamodell erleichtert somit die Fehlerprävention, indem es ungültige oder unvollständige Modelle bereits bei der Erstellung unterbindet. So werden Fehler frühzeitig erkannt oder gänzlich verhindert, was die Qualität der resultierenden Systeme steigert.

Die Metamodellierung besteht aus vier hierarchisch angeordneten Ebenen:

Auf der M3 Ebene befindet sich die Metamodelling Language, die Sprache zum Bauen von Sprachen. Sie definiert die Mittel, mit denen die abstrakte Syntax in einem Metamodell auf der M2 Ebene festgelegt werden kann.

Eine MML besteht aus verschieden Elementen:

  • Meta-Class: Abstrakte Bausteine, aus denen die Konstrukte der M2-Ebene gebildet werden, also legt sie fest welche Arten von Elementen genutzt werden dürfen. Beispiele für MetaClass wären Class (für Entitäten), Attribute (für Eigenschaften), Association (für Beziehungen), DataType (für Datentypen).
  • Meta-Attribute: Eigenschaften von MetaClass und für jede Instanz auf der M2 Ebene relevant sind. Beispiele hierfür sind beispielsweise name (des Attributes auf M2) : String, type : DataType
  • Meta-Associations: Geben an wie Metaklassen miteinander verknüpft werden können oder müssen. Die Meta-Class „Class“ ist mit der „Attribute“ mit einer „hat“ Verknüpfung verbunden, und zwar so, dass eine Class mehrere Attribute haben kann

Diese werden benötigt, um Konstrukte des Modells auf der M2 Ebene, ihre Kombinationsregeln sowie deren Semantik festzulegen, sodass modellbasierte Werkzeuge die später erstellten Modelle korrekt interpretieren können. Diese Ebene ist somit essenziell für die Entwicklung von DSMLs.

Die M2 Ebene enthält das Metamodell, also die DSML, und legt damit die abstrakte Syntax dieser Sprache fest. Sie definiert, welche Elemente in einem Modell vorhanden sein dürfen, welche Eigenschaften diese besitzen und wie sie zueinander in Beziehung stehen können. Zusätzlich können auf dieser Ebene Regeln (Constraints) formuliert werden, um die Konsistenz und Widerspruchfreiheit der Modelle sicherzustellen.

Die M1 Ebene beinhaltet die konkreten Modelle, die mithilfe der auf der M2-Ebene definierten Modellierungssprache erstellt werden. Die Modelle sind domänenspezifisch und bilden Strukturen, Prozesse oder Systeme ab, wobei sie die im Metamodell festgelegten Elemente, Eigenschaften und Beziehungen umsetzen. Die auf der M2 Ebene festgelegten Syntax- und Konsistenzregeln stellen sicher, dass die Modelle korrekt und widerspruchsfrei sind. Auf dieser Ebene findet somit die konkrete Modellierung statt, somit werden die abstrakten Konzepte und Einschränkungen mit inhaltlichen Daten gefüllt, um reale Szenarien abzubilden.

Die M0 Ebene umfasst konkrete Instanzen, Objekte oder Ereignisse aus der realen Welt, die den auf M1 definierten Modellen entsprechen. Jedes Modell auf dieser Ebene ist eine Umsetzung der in M1 spezifizierten Strukturen oder Prozesse und besitzt konkrete Werte für alle im Modell vorgesehenen Attribute. Somit enthält diese Ebene reale Ausprägungen der modellierten Elemente einschließlich ihrer physischen Eigenschaften und tatsächlichen Beziehungen, wie sie in der beobachtbaren Realität vorkommen.

Zur Veranschaulichung der Metamodellierung ziehen wir eine Alltagsmetapher als Beispiel hinzu. In einem Modellierungsprozess eines Autos, würde die Metamodellierungssprache auf der M3-Ebene festlegen, dass wir in der darunterliegenden Ebene, Entitäten selbst erstellen können, welche Attribute haben und miteinander in Beziehung stehen. Auf der M2-Ebene wird darauf aufbauend ein Metamodell für eine spezifische Fahrzeugklasse, wie einem Kleinwagen, definiert.

Dieses gibt vor, dass ein Wagen dieser Klasse einen Motor, eine Karosserie, vier Räder aber auch Eigenschaften wie eine Farbe, Motorleistung und Gewicht besitzt. Der Bauplan eines konkreten Modells, wie einem VW Golf 8, entsteht auf der M1-Ebene. Das Abbild der Realität erhält inhaltliche Details wie, eine Motorleistung von 110 kW, die Farbe Blau und vier 17-Zoll Felgen. Das reale Auto wird befindet sich dann auf der Ebene M0. Es handelt sich hierbei um ein tatsächliches Auto, welches sich in jemandes Besitz befinden könnte.
